# Docker 部署

如果你希望在自己的机器或服务器上运行主应用，可以直接使用 Docker Compose。镜像会构建 `web/` Vite 静态产物，用 nginx 提供页面，并启动一个 Go 服务提供 `/api/proxy` 同域 AI 请求代理。

## 使用 Docker Compose

```bash
git clone git@github.com:cangerx/c-aihuabu.git
cd c-aihuabu
docker compose up -d
```

启动后访问：

```text
http://localhost:3000
```

## 本地构建镜像

如果需要使用本地镜像名：

```bash
docker compose -f docker-compose.local.yml up -d --build
```

## 文档站镜像

文档站位于 `docs/` 目录，按带服务端能力的 Next.js standalone 应用单独构建，不打进主应用镜像。

使用发布镜像：

```bash
cd docs
docker compose up -d
```

基于当前源码本地构建：

```bash
cd docs
docker compose -f docker-compose.local.yml up -d --build
```

## 数据说明

当前主应用镜像提供静态文件和可选 AI 请求代理。画布、我的素材、生成记录和 AI API Key 默认保存在浏览器本地；第三方提示词由浏览器直拉 GitHub Raw 后缓存在本地 IndexedDB，不需要额外挂载数据目录。

Docker 运行时环境变量不会注入已构建的前端页面；需要变更文档链接等前端变量时，请在构建前使用 `VITE_*` 环境变量。镜像内置 `/api/proxy` 转发 AI 上游请求，以及 `/api/uploads/references` 参考素材临时上传（默认 15 天清理）；不提供账号后端或 WebDAV 转发。参考素材公网地址依赖 `C_AI_PUBLIC_BASE_URL` 与上传目录挂载。
