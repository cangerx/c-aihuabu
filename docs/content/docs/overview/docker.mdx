# Docker 部署

如果你希望在自己的机器或服务器上运行主应用，可以直接使用 Docker Compose。镜像会构建 `web/` Vite 静态产物，并用 nginx 提供页面。

## 使用 Docker Compose

```bash
git clone git@github.com:cangerx/c-aihuabu.git
cd c-aihuabu
docker compose up -d
```

启动后访问：

```text
http://localhost:3000
```

## 本地构建镜像

如果需要使用本地镜像名：

```bash
docker compose -f docker-compose.local.yml up -d --build
```

## 文档站镜像

文档站位于 `docs/` 目录，按带服务端能力的 Next.js standalone 应用单独构建，不打进主应用镜像。

使用发布镜像：

```bash
cd docs
docker compose up -d
```

基于当前源码本地构建：

```bash
cd docs
docker compose -f docker-compose.local.yml up -d --build
```

## 数据说明

当前主应用镜像只提供静态文件。画布、我的素材、生成记录和 AI API Key 默认保存在浏览器本地；第三方提示词由浏览器直拉 GitHub Raw 后缓存在本地 IndexedDB，不需要额外挂载数据目录。

Docker 运行时环境变量不会注入已构建的前端页面；需要变更文档链接等前端变量时，请在构建前使用 `VITE_*` 环境变量。静态版不包含账号后端、AI 请求代理、服务端临时上传和 WebDAV 转发。
