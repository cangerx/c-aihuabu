# 待测试

## 本轮重点测试

- Vite 静态前端迁移：`web/` 应通过 `bun run dev` 启动 Vite，通过 `bun run build` 生成 `web/dist`；首页、生图、视频、素材、提示词、画布列表、画布详情和 404 路由都应由 React Router 正常打开，刷新深层路由时 Docker/nginx 应回退到 `index.html`。
- 静态版运行边界：配置页“请求方式”应显示“浏览器直连 / 同域代理”；升级后默认应选中同域代理，AI 请求应走 `/api/proxy?url=...`；手动切回浏览器直连时才直接请求上游；账号/云端个人渠道操作应提示静态版不包含账号后端；提示词页应由浏览器直拉 GitHub Raw 并写入本地缓存；WebDAV 只支持前端直连；Cai / Seedance / Lingdong / NewToken 等异步视频参考素材使用本地文件时应提示先上传到公网 HTTPS 地址。
- Docker 静态镜像：`docker compose up -d --build` 应构建 Vite 产物并用 nginx 监听 3000，同时启动 Go 代理监听容器内 8787；`/healthz` 应返回 `ok`，`/api/proxy` 应能转发公网 http/https 上游，静态 JS/CSS 响应应支持 gzip。
- 重构前核心功能盘点文档：新增 `docs/content/docs/backend/core-capability-inventory.mdx`，梳理当前页面、核心领域对象、本地持久化位置、AI 调用链、服务端能力、性能瓶颈和高性能重构边界；文档索引和开发文档导航应能看到“核心功能盘点”入口。
- Grok Imagine 新模型：选择 `grok-imagine-image-quality` / `grok-imagine-image` 时，图像设置应切换为 `1k/2k` 和对应宽高比，提交 `/v1/images/generations` 与 `/v1/images/edits` 时应发送 `aspect_ratio` / `resolution` JSON；选择 `grok-imagine-video` / `grok-imagine-video-1.5` 时，视频设置应切换为 `480p/720p/1080p` 和对应宽高比，`grok-imagine-video-1.5` 在画布里不应再显示“图片参考”模式，只允许单图图生视频，1080p 只在允许的模型/模式下提交，多图编辑应按 `images` 数组提交且超过 3 张要直接报错。配置页“填入 Cai 视频模型”也应包含 `grok-imagine-video-1.5`，避免默认模型列表漏项。
- Grok Imagine 图片结果落盘：Grok 图片生成/编辑请求应优先使用 `response_format=b64_json`，生图工作台和画布节点拿到结果后应转成本地可持久化资源；历史记录或第三方渠道仍返回外链图片 URL 时，应在浏览器 CORS 允许的情况下转存，转存失败也要保留原始 URL 用于显示。
- 生图工作台参数同步：桌面和移动端的图像设置面板都应按当前选中的模型切换到对应的 StepFun 固定尺寸或 Grok 1k/2k + 宽高比，摘要栏、设置面板和提交请求必须使用同一份归一化配置。
- 主题初始化脚本清理：打开任意页面时控制台不应再出现 `Encountered a script tag while rendering React component`；切换浅色/深色主题后刷新页面，`index.html` 内联脚本应根据本地主题缓存提前设置 `html.dark` 和 `color-scheme`。
- StepFun `step-image-edit-2`：选择该模型时，图像设置应切换为文档里的固定尺寸 `1024x1024`、`768x1360`、`896x1184`、`1360x768`、`1184x896`；提交 `/v1/images/generations` 时应自动补上 `cfg_scale=1.0`、`steps=8` 和选中的 `size`，提交 `/v1/images/edits` 时也应走该模型的专用落参，尺寸显示和节点比例要同步归一化，尺寸卡片预览也不能再报 `SizePreview is not defined`。
- 同步生图请求方式：关闭同域代理时，同步 `/v1/images/generations` 或 `/v1/images/edits` 应直接请求用户配置的 Base URL；开启同域代理时应请求同域 `/api/proxy`，浏览器不应再出现上游 CORS 报错。
- 同步图片直连稳定性：静态版不再产生 `[ai-proxy-task]` 服务端日志；Network 中不应出现 `/api/ai-proxy` 请求。
- 前端缓存更新：部署新版本后重新打开页面，Network 里不应再出现旧 `/api/ai-proxy`、`/api/uploads` 或 `/webdav-proxy` 请求；如果旧页面尝试加载已不存在的动态 JS chunk，应由 nginx fallback 或路由错误兜底自动刷新一次并恢复到新入口，不应显示 React Router 默认错误页。
- 对应提交范围：`d6b430f` 起到当前最新提交，重点覆盖脚本生成器、参考图片托管和视频提交链路。
- 画布脚本生成器：新建“脚本生成器”节点，测试三种模式“主题分镜 / 参考图文案 / 图文转视频”；点击节点内“上传参考图”后，应自动创建图片节点并连线到脚本生成器；只上传图片不输入文字时，也应能生成文案分析摘要和分镜卡片；点击“拆分镜头”后，应生成“镜头文本 -> 图片节点 -> 视频节点”的工作流。
- 视频参考图片提交：从画布图片节点连接到视频节点后提交时，Cai / NewToken / duomiapi / Seedance / Lingdong 等需要服务端回源的异步接口应只接受公网 HTTPS URL；本地 `blob:` 或 dataURL 应在前端直接提示先上传到对象存储。
- Grok 1.5 预览图生视频：选择 `grok-imagine-video-1.5-preview` 并连接 1 张图片后提交，Cai 专用接口、NewToken 和 duomiapi payload 应使用 `input_reference` 提交参考图，不应再因缺少 `input_reference` 返回 `requires one input_reference image`。
- 参考素材公网读取：用户手填的公网 HTTPS 参考素材应能通过 `HEAD` 或 `GET Range` 自检；静态版不再提供 `/api/uploads/references` 文件读取接口。
- 参考素材强制公网校验：Cai、NewToken、Duomi、Lingdong 和 Seedance 视频参考素材提交前必须是可公网读取的 HTTPS URL，并通过 `HEAD` 或 `GET Range` 自检；本地素材不再尝试上传到本服务。
- 生图/视频工作台并发提交：生图工作台和视频创作台在已有任务生成中时，提交按钮不再锁死，可继续提交新的生成任务；历史记录应保留每次提交，视频任务会继续后台轮询多个任务；视频创作台需重点确认连续点击提交不会被旧 `setRunning` 状态阻断。
- 全能参考模式：支持全能参考的模型连接图片/视频素材时应能切换到“全能参考”；只连接音频素材时应提示需要配合图片或视频素材，避免提交空参考 payload。
- Cai API `videos` / `videos_stable`：配置页选择 Cai API 后可一键填入 Cai 视频模型；选择 `videos` 或 `videos_stable` 时，画布视频节点应显示并支持首尾帧、全能参考和参考视频，提交 payload 应走 `metadata.media`。
- 引用素材交互：画布视频节点连接音频后，参考缩略图应显示音频图标；点击画布输入框、生图工作台或视频工作台提示词框的 `@` 按钮，应立即弹出当前连接/已添加参考素材的候选菜单，不必手动输入 `@`；生图工作台提交时，已添加的参考图必须随请求进入 `requestEdit` 并作为图片文件传给上游，视频工作台提交时参考图、参考视频、参考音频也必须随请求进入 `createVideoGenerationTask`，不只是前端展示标签。
- 引用素材提交语义：生图和视频提交前应把 `@图片1`、`@视频1`、`@音频1` 规范成 `图片1`、`视频1`、`音频1`，并在提示词前追加“参考图片/素材编号”说明；Cai、NewToken、Duomi、OpenAI 兼容视频和 Seedance 格式都应收到编号说明和实际参考素材字段，不能只提交前端 `@` 展示文本。
- 视频生成记录缩略图：视频创作台生成成功后应自动截取视频首帧作为历史记录缩略图；历史记录卡片应显示视频封面，旧记录或截帧失败时应回退显示视频首帧预览；加入“我的素材”的视频也应带封面并在刷新后继续显示。
- 视频生成记录时间：视频创作台左侧生成记录卡片应显示生成时间；旧记录若缺少 `time` 字段，应回退使用 `createdAt` 格式化展示。
- 失败/生成中任务清理：生图工作台和视频创作台生成中或上游已失败的历史记录应可选中后点击“停止”结束本地任务，也可直接删除；删除后后台异步请求或视频轮询不应再把同一条记录重新写回。画布里删除正在生成的节点时，关联生成请求也应被中断，不能在节点删除后又写回结果。
- NewToken 参考素材限制：静态版不再内置 `/api/uploads/newtoken-references` 媒体中转上传；NewToken 异步生图/视频参考素材需要用户提供公网 HTTPS URL。
- 在线 Agent 对话：画布右侧 Agent 使用文本模型聊天并调用工具时，如果上游不兼容 Responses 工具 schema，应自动降级到 Chat Completions 工具格式继续请求，不应再出现 `Bad input` / `tools/0/function` / `enum function not in custom` 这类错误。
- Docker 本地编译部署：默认 `docker-compose.yml` 应从当前目录 Dockerfile 构建 `c-aihuabu:local` 静态镜像；线上更新时执行 `git pull` 后直接 `docker compose build --no-cache && docker compose up -d --force-recreate`，不再依赖远端 `latest` 镜像缓存。

- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- 调用格式显示：配置弹窗、云端渠道摘要和相关错误提示中，调用格式不再按“官方接口 / 第三方接口”分组，Cai、NewToken、Duomi 不再显示 API 字样，Key 相关提示也不再显示 API Key。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。
- 音频和视频生成：选择 Gemini 调用格式时先给出不支持提示，仍需使用 OpenAI 兼容渠道。
- 画布脚本生成器：工具栏和右键菜单新增“脚本生成器”节点，可根据输入需求和上游参考内容调用文本模型生成结构化分镜；节点内展示镜头卡片，并支持一键拆分为“镜头文本 -> 图片节点 -> 视频节点”的批量图片/视频工作流。脚本节点新增“主题分镜 / 参考图文案 / 图文转视频”模式切换和“上传参考图”入口，上传图片会自动创建图片节点并连线到脚本生成器；有上游图片时会先生成画面与文案分析摘要，再生成图片提示词和视频脚本。
- 账号与云端个人渠道：静态前端版本不包含账号后端；配置弹窗账号 Tab 的登录、注册、保存云端渠道应给出不可用提示，本地渠道配置仍应正常保存到浏览器。
- 高并发任务系统底座：静态前端版本已移除 PostgreSQL/Redis 依赖和服务端 schema 草案；后续高性能重构需要在独立后端重新设计用户、渠道、任务、队列和对象存储。
- 画布图片去元数据工具：图片节点悬浮工具栏新增“去元数据”，会把当前图片通过 Canvas 重编码为 PNG，移除 EXIF/GPS/软件信息等文件元数据，并做轻微像素扰动以弱化简单文件哈希指纹；该功能不承诺对抗专业感知哈希或水印识别。
- 无限画布复制图片：在图片节点悬浮工具栏新增“复制图片”功能，支持将图片（自动转换为 PNG）复制到系统剪切板，可直接在其他应用中粘贴或在画布中 Ctrl+V 粘贴新建节点。
- Cai 专用接口：渠道设置中的原“OpenAI (JSON)”显示改为“Cai 专用接口”；`happyhorse` / `grok` 等模型按 C-AI 文档走 `POST /v1/videos` 和 `GET /v1/videos/{task_id}`，`seedance` / SD 模型走兼容路径 `POST /v1/video/generations` 和 `GET /v1/video/generations/{task_id}`；参考图片、视频、音频会要求使用公网 URL，避免本地 blob/dataURL 提交到上游导致 408 超时。
- 异步视频接口兼容：OpenAI 风格视频任务创建兼容 `id`、`task_id`、`taskId` 等任务 ID 字段；轮询兼容 `completed`、`succeeded`、`success`、`done` 等完成状态，并能从 `url`、`video_url`、`content.video_url`、`output.url`、数组型 `output` 等常见字段读取视频地址，降低三方聚合接口返回结构差异导致的失败概率。
- 图片生成记录恢复：生图工作台在发起同步图片请求前会先保存“生成中”记录，并在每张图片成功后立即写入本地记录。页面刷新或网络中断后，历史记录中可继续看到已完成的图片和未完成占位，避免整批结果丢失；未完成图片仍需重新发起请求生成，因为同步图片接口本身没有可恢复的任务 ID。
- 视频任务恢复：视频创作台在异步任务创建成功后会把任务 ID 保存到本地记录。轮询阶段如果网络中断或临时查询失败，不再直接把任务标记为失败，而是保留“生成中”状态并显示轮询暂时中断提示；浏览器恢复联网、页面重新回到前台或点击生成中的历史记录时，会继续查询同一个任务。
- 画布视频手动拉取：画布视频节点在异步任务创建成功后会保存上游任务 ID。若后续轮询超时、页面刷新或网络中断，错误节点会显示“拉取结果”按钮，也可通过节点悬浮工具栏手动查询同一个任务；任务已完成时会写回视频文件，仍在生成时保留可再次拉取的错误态。
- 视频长任务轮询：画布和视频创作台的视频任务轮询最长等待时间调整为 30 分钟，适配上游已返回任务 ID 但生成耗时较长的情况。
- 视频成功结果显示：当异步视频轮询结果已经返回公网视频 URL 时，画布和视频创作台应直接显示该 URL，不再等待前端完整下载视频文件后才更新成功状态；上游已成功到页面可见结果的延迟应接近轮询间隔。
- Lingdong 调用格式：配置页新增 Lingdong 渠道格式，默认 Base URL 为 `https://www.lingdongapi.com`，可一键恢复 `gpt-image-2`、`sora-2` 和 `sd-2-*` 模型；图片生成应走 `/v1/images/generations`，视频创建和轮询都应走 `/v1/video/generations`，并能读取 `url` / `video_url` / `result_url` / `content_url` 返回结果。
- 视频上传反馈：视频创作台在选择参考图、参考视频或参考音频后，上传按钮和结果区应显示“正在上传”忙碌态，不再给用户空白等待的感觉；上传完成后再恢复可操作状态。
- 火山方舟 (Seedance) 专属格式：保留“火山方舟 (Seedance)”调用格式选项，当直接对接火山引擎官方方舟 Agent Plan 时，系统会使用其特有的 `/contents/generations/tasks` 异步视频生成与轮询接口。
- 生成输入框关联与光标修复：当生成节点有相连参考素材时，输入框的占位文本会增加 `，按 @ 引用连接的图片或视频` 的引导提示。同时修复了由于富文本高亮遮罩层将 textarea 的文字颜色设为透明，导致 WebKit (Chrome/Safari) 浏览器下闪烁的输入光标 (Caret/输入标) 一并消失、无法正常点击聚焦的问题（现改为 textarea 保持完全标准的前景色与光标颜色，将透明层逻辑下沉到底部的高亮遮罩层中，实现完全原生的输入标提示与聚焦反应）。
- 提示词中心 UI 优化与标签清洗：重构提示词中心页面布局，设计了带有微光渐变的毛玻璃头部 Banner，并优化了搜索框与清除样式；分类过滤选项由原本的英文仓库 ID 映射为更友好的中文名称，并采用极简贴合的主题分段控制选项；标签过滤经过严谨的规则清洗，自动屏蔽了杂乱的作者用户名（如 `@` 开头的推特账号）和无效字符，并将常见英文或混乱词条合并标准化（如 `portrait` / `portraits` 映射为“肖像”，`3d render` 映射为“3D渲染”），采用轻量的药丸胶囊状过滤按钮，超出的标签支持一键折叠与展示，使界面极为整洁清爽；同时，卡片设计重构为支持图像 Hover 缩放动画与流畅圆角阴影的定制布局，大幅提升了黑白主题下的视觉高级感；该优化已同步适配至画布内的“提示词库”选择弹窗与提示词详情展示弹窗。

- 输入框禁用回车生成：去除了画布生成输入框中按回车键（Enter）触发内容生成的逻辑，现在只支持通过点击右下角按钮手动提交生成，回车键在输入框中恢复为插入换行的默认行为。
- 生成后保留提示词：修改了画布生图面板的提交行为，点击生成后不再自动清空输入框内的提示词，方便用户在原有提示词基础上微调并快速重新提交。
- **模型拉取请求方式**：关闭同域代理时“拉取模型列表”直接请求渠道 `/models`；开启同域代理时应经 `/api/proxy` 拉取，渠道如果自身返回鉴权或模型错误，应显示上游错误并允许用户手动填写模型。
- **首页动画与视觉效果优化**：首页（`/`）重构为动态交互式设计，增加了平滑浮动的极光氛围渐变光晕（Glow Blobs），采用 Framer Motion 实现所有核心文案与按钮的序列淡入与上滑。同时，副标题高亮划线支持随字显现后横向延迟伸展划过。
- **画布创意连线节点演示墙 (CanvasPreview)**：在首页下方增设了一个动态浮动的微缩节点画布演示区，包括“提示词节点”、“生图节点”（带流光渐变极光完成态）与“视频节点”（带播放组件和滚动进度条），这三个卡片在大屏下通过 SVG 折线连结，连接线具有淡入成线与虚线亮斑数据流动的视觉效果，完美演绎了 C-AI 画布“生成、连接和重组，从单次生成到连续推演”的产品理念。
- **C-ai Agent 基础使用说明**：画布侧边栏的连接页补充了 C-ai Agent 与 MCP 的基础说明，分开说明网页侧边栏、Codex 快速接入和通用 MCP Agent 接入三种路径；爱马仕 Agent、OpenClaw、Claude Code 等支持 stdio MCP 的客户端都可以通过 `npx -y c-ai-agent mcp` 接入画布工具。
- **画布统一输入框光标修复**：修复了生成后再次编辑节点时，统一输入框因引用高亮覆盖层把真实 textarea 文本设为透明，导致输入光标不可见的问题；现在输入框聚焦时优先使用原生 textarea 显示，保证光标稳定。
- **画布 @引用菜单定位修复**：统一输入框输入 `@` 时，图片/视频/文本引用候选菜单改为按当前输入光标位置定位，并限制在可视区域内，避免菜单随着画布浮层、缩放或多行输入出现乱飘。
- **画布生成后保留提示词修复**：统一输入框生成出图片、文本、视频或音频结果后，会继续保留原来的文案提示词；继续编辑输入框时也会同步写回节点，避免面板重渲染后丢失。
- **视频参考图片公网 URL**：Cai 专用视频接口创建任务前，本地参考图片不再上传到服务端；需要先放到对象存储或其他公网 HTTPS 地址，再提交给上游。
- **视频参考媒体公网 URL**：Cai / NewToken / duomiapi 视频接口创建任务前，参考视频和参考音频必须是公网 HTTPS 地址；本地文件会提示先上传到对象存储。
- **视频参考素材公网读取增强**：Cai/NewToken/duomiapi/Lingdong/Seedance 提交参考素材前会对用户提供的公网 HTTPS URL 做可访问性自检；当 `HEAD` 不可用时会改用 `GET Range` 探测兜底。
- **772 视频渠道自动兼容 Cai**：当视频渠道 Base URL 使用 `ai.772.ee` / `api.772.ee` / `*.772.ee` 时，即使渠道调用格式未手动切到“Cai 专用接口”，也会自动使用 Cai JSON 视频任务创建逻辑并上传本地参考图片，避免误走 OpenAI multipart 文件直传导致 500。
- **NewToken 异步视频接口适配**：渠道调用格式新增 `NewToken 异步接口`，配置页支持一键填入常用视频模型；视频生成会按 `POST /v1/videos` / `GET /v1/videos/{task_id}` 协议创建和轮询任务，并按模型族适配 `video-standard-720p`、`video-pro-720p`、`video-fast-720p`、`sora-2`、`sora-vip3-pro-*`、`veo-omni-flash`、`veo-omni-flash-video-edit`、`veo-3-1` 的参考图/视频/音频字段。
- **NewToken gpt-image2 异步生图适配**：NewToken 调用格式下，`gpt-image2-1k` / `gpt-image2-2k` / `gpt-image2-4k` 会按官方文档改走 `POST /v1/videos` 创建异步图片任务，并轮询 `GET /v1/videos/{task_id}` 读取 `image_url` / `url` / `metadata.result_urls` 结果；只有 `gpt-image2-*_sync` 模型继续走同步 `/v1/images/generations`。
- **duomiapi 视频接口适配**：渠道调用格式新增 `duomiapi`，默认 Base URL 为 `https://duomiapi.com`，配置页内置当前已适配的 Seedance / Grok 视频模型，不再依赖拉取模型；视频生成按 duomiapi 文档分别适配 Seedance 官方格式 `/api/v3/contents/generations/tasks` 和 Grok `/v1/videos/generations` 创建/查询路径。
- **视频最大生成时长限制**：通用视频设置面板的快捷秒数和手动输入上限统一限制为 15 秒，避免选择 16 秒或 20 秒这类上游视频模型不支持的时长。
- **画布图片粘贴修复**：画布支持通过浏览器原生 `paste` 事件读取剪贴板图片文件，修复仅依赖 `navigator.clipboard.read()` 导致在部分浏览器、非 HTTPS 或权限受限环境下复制图片后无法粘贴到画布的问题。
- **画布剪贴板图片兼容增强**：画布和 C-ai Agent 输入框粘贴图片时同时读取 `clipboardData.files` 与 `clipboardData.items`，兼容截图工具、浏览器图片复制、系统剪贴板等不同来源，避免图片存在于剪贴板但无法创建节点。
- **参考图片 HTTPS 公网地址**：静态版不再支持服务端上传接口；视频参考图片必须由用户提供 HTTPS 公网地址，避免上游拒绝拉取本地或 plain `http://` 链接。
- **画布自动整理排版重叠与视口对焦优化 (Tidy Up)**: 深度优化了自动整理画布的排版算法。列间距从 `100` 宽至 `280` 像素，行间距从 `60` 宽至 `100` 像素，大大舒缓了节点间连线倾角，避免重叠与视觉挤压；同时，**新增了自动视口对焦居中逻辑**：整理完成后，算法自动计算所有节点的总包围盒（Bounding Box），计算出最适宜的缩放比 `k`（介于 `0.25` 到 `1.0` 之间）和画布位移 `x, y` 并自动应用到视口，使用户无需手动缩放、平移即可将整理后的树状推演图完美呈现在屏幕中央。
- **视频生成面板与有向图连流动效升级**：重构了画布中视频生成节点的连线反馈与配置面板交互：
  - **连线流动特效 (Flowing Glow)**：当任何图片或视频节点连线至本视频节点时，有向折线上将增加沿方向匀速滑动的蓝色光粒流向动效，表示参考素材处于活动数据传输状态。
  - **参考模式 Tab 栏**：新增 `文生视频`、`图生视频`、`首尾帧`、`图片参考` 等模式，并根据当前模型和已连入素材动态控制可用状态；Seedance / doubao-seedance、NewToken 部分视频模型和 `veo-3-1` 会按能力显示“全能参考”，普通 Grok 视频模型只保留文生/图生/图片参考，`grok-imagine-video-1.5-preview` 会要求先连接图片后再提交。
  - **Cai 视频 payload 按模型能力拆分**：Seedance `首尾帧` 会提交 `metadata.media` 的 `first_frame` / `last_frame`，`全能参考` 会提交 `reference_image` / `reference_video` / `reference_voice`；普通图片参考不再误写 `first_image` / `last_image`，避免上游把多图参考误判为首尾帧。
  - **输入框内快捷辅助与缩略图列表**：在输入框内上方排布 `标记`、`特效`、`角色库`、`+ 参考` 四个精致的胶囊按钮。后面会实时显示当前已连线的参考素材图片/视频方形缩略图（带蓝色数字角标），当鼠标悬浮在缩略图上时角标会翻转为红色 `×` 关闭按钮，点击即可实时断开与该参考节点的折线连接，动效和缩略图同步清除。
  - **“运镜” (Camera) 下拉配置**：在面板下方配置了带有摄像机图标的运镜参数下拉选项，选择后会自动在提示词文末注入/同步对应的运镜指令，并支持从文本中反向同步选择状态。
  - **视频任务状态卡片**：视频节点生成中时会显示专业任务卡片，包含模型、任务 ID、参考素材数量、模式、时长、尺寸、清晰度和运镜等信息；失败态会优先展示任务 ID 和“拉取结果”按钮，便于上游任务已完成但浏览器轮询中断时手动恢复；成功态视频底部会保留模型、模式、时长、参考数量、运镜和任务 ID 信息层。
  - **像素级极简单行底栏与魔法棒优化**：完全重构了提示词面板的外观与排版：
    - 外层面板在浅色模式下升级为纯白 `#ffffff` 大圆角卡片并带极轻投影；
    - 输入区域去除了双重边框嵌套，与面板融为一体，极为舒展；
    - 底部重构为扁平单行排版：最左侧为扁平的 `+`（唤起提示词库）与 `@` 按钮（一键在光标处增加 `@` 引用符号并聚焦）；
