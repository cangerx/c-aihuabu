# 待测试

- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。
- 音频和视频生成：选择 Gemini 调用格式时先给出不支持提示，仍需使用 OpenAI 兼容渠道。
- 账号与云端个人渠道：配置弹窗新增“账号”Tab，支持邮箱密码注册/登录；登录后可把本地模型渠道保存为云端个人渠道，API Key 使用服务端 `AI_HUABU_SECRET_KEY` 加密后写入本地服务数据目录。云端渠道当前用于多端配置保存底座，后续异步任务 Worker 接入后会使用服务端密文调用上游；密钥不会下发回浏览器。
- 高并发任务系统底座：新增 PostgreSQL/Redis 依赖和服务端 schema 草案，覆盖 `users`、`sessions`、`model_channels`、`generation_tasks`、`generation_task_items`，为后续图片同步接口后端任务化、Worker 队列、对象存储结果落盘做准备。当前账号 API 仍是开发态文件存储实现，生产高并发需要迁移到 `DATABASE_URL` + `REDIS_URL`。
- 画布图片去元数据工具：图片节点悬浮工具栏新增“去元数据”，会把当前图片通过 Canvas 重编码为 PNG，移除 EXIF/GPS/软件信息等文件元数据，并做轻微像素扰动以弱化简单文件哈希指纹；该功能不承诺对抗专业感知哈希或水印识别。
- 无限画布复制图片：在图片节点悬浮工具栏新增“复制图片”功能，支持将图片（自动转换为 PNG）复制到系统剪切板，可直接在其他应用中粘贴或在画布中 Ctrl+V 粘贴新建节点。
- Cai 专用接口：渠道设置中的原“OpenAI (JSON)”显示改为“Cai 专用接口”；`happyhorse` / `grok` 等模型按 C-AI 文档走 `POST /v1/videos` 和 `GET /v1/videos/{task_id}`，`seedance` / SD 模型走兼容路径 `POST /v1/video/generations` 和 `GET /v1/video/generations/{task_id}`；参考图片、视频、音频会要求使用公网 URL，避免本地 blob/dataURL 提交到上游导致 408 超时。
- 异步视频接口兼容：OpenAI 风格视频任务创建兼容 `id`、`task_id`、`taskId` 等任务 ID 字段；轮询兼容 `completed`、`succeeded`、`success`、`done` 等完成状态，并能从 `url`、`video_url`、`content.video_url`、`output.url`、数组型 `output` 等常见字段读取视频地址，降低三方聚合接口返回结构差异导致的失败概率。
- 图片生成记录恢复：生图工作台在发起同步图片请求前会先保存“生成中”记录，并在每张图片成功后立即写入本地记录。页面刷新或网络中断后，历史记录中可继续看到已完成的图片和未完成占位，避免整批结果丢失；未完成图片仍需重新发起请求生成，因为同步图片接口本身没有可恢复的任务 ID。
- 视频任务恢复：视频创作台在异步任务创建成功后会把任务 ID 保存到本地记录。轮询阶段如果网络中断或临时查询失败，不再直接把任务标记为失败，而是保留“生成中”状态并显示轮询暂时中断提示；浏览器恢复联网、页面重新回到前台或点击生成中的历史记录时，会继续查询同一个任务。
- 画布视频手动拉取：画布视频节点在异步任务创建成功后会保存上游任务 ID。若后续轮询超时、页面刷新或网络中断，错误节点会显示“拉取结果”按钮，也可通过节点悬浮工具栏手动查询同一个任务；任务已完成时会写回视频文件，仍在生成时保留可再次拉取的错误态。
- 视频长任务轮询：画布和视频创作台的视频任务轮询最长等待时间调整为 30 分钟，适配上游已返回任务 ID 但生成耗时较长的情况。
- 火山方舟 (Seedance) 专属格式：保留“火山方舟 (Seedance)”调用格式选项，当直接对接火山引擎官方方舟 Agent Plan 时，系统会使用其特有的 `/contents/generations/tasks` 异步视频生成与轮询接口。
- 生成输入框关联与光标修复：当生成节点有相连参考素材时，输入框的占位文本会增加 `，按 @ 引用连接的图片或视频` 的引导提示。同时修复了由于富文本高亮遮罩层将 textarea 的文字颜色设为透明，导致 WebKit (Chrome/Safari) 浏览器下闪烁的输入光标 (Caret/输入标) 一并消失、无法正常点击聚焦的问题（现改为 textarea 保持完全标准的前景色与光标颜色，将透明层逻辑下沉到底部的高亮遮罩层中，实现完全原生的输入标提示与聚焦反应）。
- 提示词中心 UI 优化与标签清洗：重构提示词中心页面布局，设计了带有微光渐变的毛玻璃头部 Banner，并优化了搜索框与清除样式；分类过滤选项由原本的英文仓库 ID 映射为更友好的中文名称，并采用极简贴合的主题分段控制选项；标签过滤经过严谨的规则清洗，自动屏蔽了杂乱的作者用户名（如 `@` 开头的推特账号）和无效字符，并将常见英文或混乱词条合并标准化（如 `portrait` / `portraits` 映射为“肖像”，`3d render` 映射为“3D渲染”），采用轻量的药丸胶囊状过滤按钮，超出的标签支持一键折叠与展示，使界面极为整洁清爽；同时，卡片设计重构为支持图像 Hover 缩放动画与流畅圆角阴影的定制布局，大幅提升了黑白主题下的视觉高级感；该优化已同步适配至画布内的“提示词库”选择弹窗与提示词详情展示弹窗。
- 画布生成面板布局重叠修复：修复了当模型名称或设置项（如 `自动 · 1:1(2k) · 1张`）文本偏长时，左侧控制选项与右侧红色“停止”按钮重叠的 Bug。现将左侧容器设为 `flex-1 min-w-0`，模型选择器及各个配置 Popover 均支持 flex-shrink 缩放及 max-width 上限，当空间吃紧时可平滑收缩并对超长文本进行优雅截断 (truncate)，同时统一了所有交互项的 h-10 按钮高度，在小屏下亦保持完全不重叠的精致视觉呈现。
- 中转服务 Key 按钮：将页面右上角及画布下拉菜单中的“文档”按钮，改为了指向中转服务 `https://api.772.ee` 的“中转服务 Key”链接，并配套将书本图标改为了钥匙（Key）图标。
- 首页模块清理：去除了首页底部的“沉淀每一次好结果”提示词展示卡片模块，简化了首页布局。
- 画布默认生成张数调整：将画布中生图节点默认的单次生成张数由 3 张修改为 1 张。
- 输入框禁用回车生成：去除了画布生成输入框中按回车键（Enter）触发内容生成的逻辑，现在只支持通过点击右下角按钮手动提交生成，回车键在输入框中恢复为插入换行的默认行为。
- 生成后保留提示词：修改了画布生图面板的提交行为，点击生成后不再自动清空输入框内的提示词，方便用户在原有提示词基础上微调并快速重新提交。
